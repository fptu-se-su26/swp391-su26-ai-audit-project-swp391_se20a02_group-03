# LƯU Ý QUAN TRỌNG VỀ CÁC FILE SQL TRONG THƯ MỤC NÀY (OBSOLETE)

🚨 **CẢNH BÁO:**
Các file SQL trong thư mục `obsolete/` này là các bản nháp thiết kế thủ công CŨ và ĐÃ LỖI THỜI.
Chúng **KHÔNG CÒN KHỚP** với Model thực tế của EF Core (ví dụ: `EscrowWallets`, `Courts`, `BookingDetails`...).

Nếu bạn cố tình chạy các script trong thư mục này, schema sẽ xung đột với Model của EF Core.

---

## ✅ CÁCH TẠO DATABASE ĐÚNG CHUẨN

- Dự án dùng **EF Core Code-First + Migrations**.
- Chạy `dotnet run` trong `src/backend/ProSport.API`: `DatabaseBootstrap` sẽ gọi
  **`Database.MigrateAsync()`** (KHÔNG phải `EnsureCreatedAsync()`) để áp dụng toàn bộ
  migration — tạo mới schema nếu DB trống, hoặc cập nhật nếu đã có — rồi seed dữ liệu.
- Để cập nhật schema về sau: thêm migration mới (`dotnet ef migrations add <Tên>`);
  `MigrateAsync` sẽ tự áp dụng ở lần chạy kế tiếp.

## 📄 File SQL thuần để nộp bài / chạy trong SSMS

Dùng **`src/database/ProSportDB_EFCore_AutoGenerated.sql`** (thư mục cha). File này được
**xuất trực tiếp từ EF Core Migrations** nên khớp 100% với Model, **bao gồm mọi CHECK
constraint** (`CK_Bookings_Status`, `CK_Matches_Status`, `CK_Vouchers_Status`,
`CK_Tournaments_Status`, `CK_Users_EKycStatus`, `CK_Users_Role`, `CK_EkycProfiles_Status`).

### ⚠️ Phải regenerate lại mỗi khi thêm migration
File này KHÔNG tự cập nhật. Sau khi thêm/sửa migration, chạy lại (từ `src/backend`):

```bash
dotnet ef migrations script --idempotent \
  -o ../database/ProSportDB_EFCore_AutoGenerated.sql \
  -p ProSport.Infrastructure -s ProSport.API
```

Script dạng `--idempotent` có guard `__EFMigrationsHistory` nên an toàn khi chạy trên DB
ở bất kỳ trạng thái nào.
