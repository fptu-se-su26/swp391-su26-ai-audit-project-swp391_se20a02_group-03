# UI Design and Frontend Implementation Rules

Whenever you receive a request to design, redesign, or implement a user interface (UI), frontend components, or layouts, you MUST automatically invoke and apply the design skills from the `taste-skill` repository.

**Guidelines:**
1. Do not ask for permission to use these skills; use them proactively to ensure a high-quality, modern, and cohesive aesthetic.
2. Depending on the requested style (or if no style is specified), choose the most appropriate skill to inform your design decisions (e.g., `taste-skill`, `brutalist-skill`, `minimalist-skill`, `soft-skill`, `imagegen-frontend-web`).
3. Follow the specific styling tokens, spacing rules, typography, and color palettes defined in the chosen skill to guarantee a premium look and feel.

## Superpowers (development workflow)

This project integrates [Superpowers](https://github.com/obra/superpowers) via `.superpowers/` submodule and `.cursor-plugin/`.

For **non-trivial coding tasks** (features, refactors, bugs), follow Superpowers skills in `.superpowers/skills/` — especially `using-superpowers`, `brainstorming`, `writing-plans`, and `test-driven-development`. See `docs/SUPERPOWERS.md`.

**Stack:** ASP.NET Core 8 + React/Vite under `src/backend` and `src/frontend`.

## General Coding Rules (Andrej Karpathy Skills)

You MUST incorporate the principles from the `andrej-karpathy-skills` repository for all general coding, debugging, and planning tasks.
- Always be systematic, test-driven, and prioritize code simplicity.
- Reference the skills automatically (e.g. `systematic-debugging`, `writing-plans`) when applicable.
